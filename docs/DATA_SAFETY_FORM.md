# 데이터 보안(Data Safety) 양식 — 작성 초안

> Google Play Console → 정책 → 앱 콘텐츠 → **데이터 보안** 에 그대로 입력할 답안.
> 작성 2026-07-01 · 근거: 현재 코드(로컬 우선, 자체 서버 없음, 구글 드라이브 백업 선택)
> 회장님은 콘솔에서 아래 표대로 클릭/선택만 하면 됩니다.

---

## 1. 데이터 수집·공유 개요 (첫 화면)

| 질문 | 답 | 근거 |
|---|---|---|
| 앱이 사용자 데이터를 **수집 또는 공유**합니까? | **예** | 권한으로 사진·위치 등에 접근. (단, 당사 서버 전송은 없음) |
| 모든 수집 데이터가 **전송 중 암호화**됩니까? | **예** | 외부 전송은 구글 드라이브(HTTPS)뿐. 그 외 전송 없음 |
| 사용자가 데이터 **삭제를 요청**할 수 있습니까? | **예** | 앱 삭제·데이터 삭제·드라이브 권한 회수로 즉시 (account-deletion.html) |

> 핵심 메시지: "데이터는 기기에 저장, 당사 서버로 전송/보관하지 않음.
> 클라우드 백업은 **사용자 본인 구글 드라이브**로의 선택적 백업."

---

## 2. 데이터 유형별 선언

각 항목마다 콘솔은 ① 수집(collected) ② 공유(shared) ③ 수집 목적 ④ 필수/선택 을 묻습니다.
**공유(shared)는 전부 "아니요"** — 제3자에게 넘기지 않습니다.
**수집(collected)** 의 정의는 "앱이 기기 밖으로 데이터를 전송"입니다. 본 앱은 기기 내 처리가 원칙이며,
**사용자가 켠 클라우드 백업만 기기 밖(본인 드라이브)으로 나갑니다.**

### (A) 사진 및 동영상 → 사진(Photos)
| 항목 | 답 |
|---|---|
| 수집됨? | **예** (선택한 클라우드 백업 시 본인 드라이브로 전송) |
| 공유됨? | 아니요 |
| 목적 | **앱 기능**(백업·복원) |
| 필수/선택 | **선택**(사용자가 백업을 켤 때만) |
| 처리 위치 | 사용자 본인 구글 드라이브(appDataFolder) |

> 백업을 켜지 않으면 사진은 기기에만 있고 전송되지 않습니다. 그래도 Play 기준상
> "전송 가능성"이 있으면 수집=예로 신고하는 것이 안전합니다.

### (B) 위치 → 대략적 위치 / 정확한 위치(Location)
| 항목 | 답 |
|---|---|
| 수집됨? | **아니요** (기기 내에서만 사용, 전송 안 함) |
| 공유됨? | 아니요 |
| 비고 | 회상 알림용 좌표를 기기에 저장만 함. 외부 전송/수집 없음 → "수집 안 함"으로 선언 가능 |

> 권한은 사용하지만(매니페스트에 FINE/COARSE), 데이터를 기기 밖으로 보내지 않으므로
> Data Safety 기준의 "수집"에는 해당하지 않습니다. 권한 사용 자체는
> 콘솔의 "권한 선언"(체크리스트 15번)에서 별도로 다룹니다.

### (C) 개인 정보 → 이메일 주소(Email)
| 항목 | 답 |
|---|---|
| 수집됨? | **예** (구글 로그인 시 본인 계정 식별을 위해) |
| 공유됨? | 아니요 |
| 목적 | **앱 기능**(클라우드 백업 계정 연결) · **계정 관리** |
| 필수/선택 | **선택**(클라우드 백업을 쓸 때만) |

> 이메일은 구글 로그인 과정에서 본인 계정 확인 용도로만 쓰이며 당사 서버에 저장하지 않습니다.

### (D) 앱 활동 / 기타
| 항목 | 답 |
|---|---|
| 앱 상호작용, 검색 기록, 설치 앱 목록 등 | **아니요** (수집 안 함) |
| 분석/크래시 SDK | **현재 없음** (Firebase Analytics·Crashlytics 미연동) |

> ⚠️ 향후 광고(google_mobile_ads)나 분석 SDK를 붙이면 이 양식을 **반드시 갱신**해야 합니다.
> v1은 광고·분석 없음 기준.

### (E) 키·나이·메모 등 건강/기록 데이터
| 항목 | 답 |
|---|---|
| 수집됨? | 사진과 동일 — 백업 켰을 때만 본인 드라이브로. 공유 없음 |
| 목적 | 앱 기능(백업·복원) |

## 3. 콘솔 입력 순서 요약 (회장님용)

1. 데이터 보안 → **시작** → "데이터 수집/공유: 예".
2. 데이터 유형 선택: **사진(Photos)**, **이메일 주소(Email)** 체크.
   (위치는 "수집 안 함"이므로 데이터 유형에서는 체크하지 않음.)
3. 각 유형: 공유=아니요, 목적=앱 기능(+이메일은 계정관리도), 필수여부=선택.
4. 보안 관행: "전송 중 암호화=예", "삭제 요청 가능=예" → 삭제 URL 입력
   (체크리스트 3번의 account-deletion 호스팅 주소).
5. 검토 후 제출.

## 4. 일관성 체크 (심사 반려 예방)

- 개인정보처리방침(privacy-policy.html)과 **모순 없어야 함** → 둘 다 "기기 저장, 서버 미보관,
  드라이브 백업은 선택" 으로 통일됨 ✅
- 매니페스트 권한(카메라·위치·사진·알림)과 **설명 일치** ✅
- 광고·분석 SDK 없음 선언 ↔ 실제 pubspec 에 미포함 ✅ (광고 붙이면 갱신 필요)
