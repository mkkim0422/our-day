📸 그날 우리 — 푸시 알림 정리

현재 코드 기준(lib/services/notifications/notification_service.dart) · 작성 2026-06-29

모든 알림은 기기 로컬에서 예약됩니다(서버·FCM 미사용, 외부로 데이터 전송 없음). 채널은 reminders / “촬영 리마인더”(중요도 High), 발송 방식은 zonedSchedule + inexactAllowWhileIdle(배터리 친화 — 정확한 분초가 아니라 약간의 지연이 있을 수 있음).

1. 알림 종류 한눈에 보기

#종류발송 기준시각 · 주기 제목 / 내용탭하면
1 주기 촬영 프로젝트의 촬영 주기 설정.
‘직접(manual)’은 알림 없음		 다음 주기 시각(기본 오전 10시)
매일·매주·격주·매월·매년·지정일		 제목: 페그별(“이번 기간 한 컷 📸” 등)
내용: “오늘 «제목» 한 컷 어때요? 같은 포즈로 그날의 우리를 남겨요.”		 촬영 화면(곧장)
2 회상(기념일) 과거 촬영 사진과 같은 월·일이 되는 날.
같은 날짜는 1건으로 묶음 · 프로젝트당 최대 8건		 그 월·일 오전 10시 · 매년 제목: “그날의 우리”
내용(1년): “작년 오늘, 가족은 이런 모습이었어요. 다시 한 컷 남겨볼까요?”
내용(N년): “N년 전 오늘의 기록이 있어요. 같은 포즈로 다시 찍어봐요.”		 추억 사진 상세 → “같은 구도로 한 컷”
3 연말 리캡 올해 촬영이 2컷 이상일 때 12/31 저녁 7시 · 매년 제목: “올해의 우리 🎞️”
내용: “«제목» 올해 기록이 모였어요. 한 해의 변화를 타임랩스로 돌아볼까요?”		 비교 · 타임랩스 화면
4a 생일 페그 이벤트 페그에 ‘생일’ 포함 + 주인공 생일 설정됨 생일 월·일 오전 10시 · 매년 제목: “오늘은 우리 기념일 🎂”
내용: “«제목» 생일이에요. 같은 포즈로 한 컷 남겨볼까요?”		 촬영 화면
4b 명절 페그 이벤트 페그에 ‘명절’ 포함 다음 설날·추석 오전 10시
2025~2035 양력 표 내장		 제목: “명절, 가족이 모인 날 🏡”
내용: “온 가족이 모인 오늘, «제목» 한 컷 어때요?”		 촬영 화면
4c 계절 페그 이벤트 페그에 ‘계절’ 포함 봄 3/1·여름 6/1·가을 9/1·겨울 12/1 오전 10시 제목: “계절이 바뀌었어요 🍃”
내용: “새 계절의 첫날, «제목» 한 컷으로 변화를 남겨요.”		 촬영 화면
5 위치 회상 위치 회상 ON + 추억 장소 반경 안에 도착 + 쿨다운(기본 6시간) 지남 즉시(앱 시작/복귀 시 위치 판정) 제목: “여기, 그때 우리 📍”
내용: “«장소»에서 찍은 추억이 있어요. 같은 구도로 다시 한 컷 남겨볼까요?”		 추억 사진 상세 → “같은 구도로 한 컷”

2. 발송 방법 · 동작 원리

예약 방식

  • 전부 로컬 예약 알림입니다(서버 푸시/FCM 없음). 비행기모드여도 예약 시각에 뜹니다.
  • 안드로이드 모드 inexactAllowWhileIdle — 배터리 절약(Doze) 영향으로 정확한 분 단위는 보장되지 않음(보통 같은 시간대 내 발송).
  • 알림별로 고유 ID 범위를 두어 서로 덮어쓰지 않음: 주기 / 회상 ×8 / 리캡 / 생일 / 명절 / 계절 / 위치회상.

언제 (재)예약되나

  • 프로젝트 생성 시, 사진 촬영(정렬 조정·백필) 시, 사진 삭제 시 → 기존 예약을 모두 취소하고 다음 1회분을 다시 예약합니다.
  • 즉 주기 알림은 “다음 한 번”만 걸려 있습니다. 앱을 가끔 열거나 촬영을 해 줘야 그다음 주기가 갱신됩니다(장기간 미사용 시 알림이 끊길 수 있음 — 개선 후보).

한도 · 끄기 · 권한

  • iOS 예약 한도(64개)를 고려해 회상 알림은 프로젝트당 최대 8건.
  • 프로젝트별로 알림 끄기 가능(scheduleConfig.push = false면 그 프로젝트 알림 전체 미예약).
  • 알림 권한(Android 13+/iOS)은 첫 진입에 강요하지 않고 사용자 흐름에서 요청.
최근 개선(2026-06-29): ① 같은 월·일 회상 알림이 여러 장이면 한 건으로 묶어 중복 발송 제거(대표는 가장 최근 사진 → “작년 오늘” 우선). ② 회상·위치 알림 탭 시 추억 사진을 먼저 보여주고 거기서 “같은 구도로 한 컷”으로 촬영 진입(전엔 바로 카메라).

3. ⚠️ 데이터 백업 — 현재 상태

지금은 클라우드 백업도, 앱 내 백업/복원 UI도 없습니다. 모든 사진·기록은 기기 안(앱 저장영역)에만 있습니다. 따라서 폰 교체 · 앱 삭제 · 기기 분실 시 데이터가 모두 사라집니다.

백업/복원 로직(로컬 ZIP)과 계정 테이블은 코드에 준비되어 있으나 화면에 연결되어 있지 않아 사용자가 쓸 수 없는 상태입니다. 설정 화면에는 “클라우드 자동 백업은 준비 중”이라고만 표시됩니다. → 안전한 보관을 위해 로그인 + 클라우드 백업(또는 최소한 로컬 백업 내보내기/복원 UI) 도입이 필요합니다.
